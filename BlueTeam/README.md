# 🔵 Blue Team Operations & Defensive Security

## 🛡️ Objetivo Defensivo
Entender la "huella" que dejan los ataques para mejorar la detección, el monitoreo y el endurecimiento (hardening) de los sistemas, aplicando protocolos oficiales de respuesta ante incidentes.

## 📖 Bibliografía de Referencia (Español)
* **"Guía Nacional de Notificación y Gestión de Ciberincidentes"** - Gobierno de España / CCN-CERT (Estándar de gestión y reporte profesional).
* **"Análisis de Tráfico de Red con Wireshark"** - Guía del INCIBE (Inspección profunda de paquetes).
* **"Gestión de Incidentes de Ciberseguridad"** - INCIBE / CCN-CERT (Respuesta y contención).

## 🛠️ Toolstack Defensivo (Laboratorio)
* **Análisis de Red:** Wireshark, Tshark.
* **Monitoreo de Logs:** Grep, Awk, Tail (Análisis de logs de Apache y System).
* **Hardening:** Guías de configuración segura basadas en estándares institucionales.

## 🗂️ Estructura de Contenidos
1. **/Traffic-Analysis:** Análisis de capturas de red (.pcap) para detectar intrusiones.
2. **/Incident-Response:** Documentación de simulacros siguiendo la Guía Nacional de Ciberincidentes.
3. **/Hardening-Guides:** Guías de mitigación para cerrar las brechas encontradas durante las fases ofensivas.
