# ☁️ Cloud Security

## 🧠 ¿Qué es Cloud Security?

La **Cloud Security** (Seguridad en la Nube) consiste en el conjunto de prácticas, tecnologías y políticas utilizadas para proteger:

- 📂 Datos  
- 🧩 Aplicaciones  
- 🖥️ Infraestructura  

dentro de entornos cloud.

---

## 🎯 Objetivo principal

- Proteger información sensible
- Evitar accesos no autorizados
- Mantener la **Confidencialidad, Integridad y Disponibilidad (CIA Triad)**

---

## 🏢 Analogía: Depósito compartido

Imaginá que guardás objetos valiosos en un depósito compartido:

- 🏢 El edificio → Infraestructura cloud  
- 🔐 Tu caja → Tus datos y aplicaciones  
- 👮 Seguridad del lugar → Proveedor cloud  
- 🔑 Tu candado → Configuración de seguridad  

👉 La seguridad depende tanto del proveedor como del usuario.

---

## ⚖️ Modelo de Responsabilidad Compartida

### ☁️ Proveedor Cloud
- Seguridad de la infraestructura
- Centros de datos
- Redes físicas
- Hardware

---

### 👤 Cliente / Administrador
- Configuración de seguridad
- Gestión de accesos
- Protección de datos
- Aplicaciones

---

## ⚠️ Riesgos comunes en Cloud

- 🔓 **Data Breach** → acceso no autorizado
- ⚙️ **Misconfiguración** → datos expuestos por error
- 🔗 **APIs inseguras**
- 🕵️ **Account Hijacking** → robo de cuentas
- 🔑 **Credenciales débiles**

---

## 🔑 Áreas clave de Cloud Security

### 🔐 Data Protection
- Encriptación:
  - En reposo (at rest)
  - En tránsito (in transit)
- Protección de datos sensibles

---

### 👤 Identity & Access Management (IAM)
- Control de acceso
- Gestión de usuarios
- Autenticación (MFA)

---

### 🌐 Network Security
- Firewalls
- VPNs
- Segmentación de red
- Protección del tráfico

---

### 📜 Compliance & Governance
- Cumplimiento de normativas
- Regulaciones (ej: GDPR, ISO)
- Políticas de seguridad

---

## 🧩 Importancia

A medida que más servicios migran a la nube:

- 📈 Aumenta la superficie de ataque
- 🔥 Crecen los riesgos
- 🛡️ Se vuelve crítica una buena seguridad

---

## 👥 Responsabilidades

### ☁️ Cloud Provider

- Monitoreo físico y lógico
- Disponibilidad del servicio

---

### 👤 Cliente
- Configuración correcta
- Control de accesos
- Protección de datos

---

### 🛡️ Equipo de Seguridad
- Estrategia de seguridad
- Evaluación de riesgos
- Implementación de controles

---

## 🔍 Testing en Cloud Security

- Evaluación de configuraciones

- Auditorías de seguridad

---

## 🔄 Mejora continua

Cloud Security requiere:

- 🔄 Actualizaciones constantes
- 🧪 Testing regular
- 📊 Monitoreo continuo
- 🎓 Capacitación de usuarios

---

## ⚠️ Punto clave

👉 La mayoría de brechas en cloud no son culpa del proveedor,  
sino de **malas configuraciones del cliente**.

---

## 🚀 Conclusión

La **Cloud Security** es fundamental en la era digital:

- Protege datos críticos
- Permite escalar de forma segura
- Reduce riesgos en entornos compartidos

👉 Es una responsabilidad compartida que requiere atención constante.
